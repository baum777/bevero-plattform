# Workflow-Mini-Audit — Gastro / Hotellerie

**Dauer:** 20–30 Minuten, im Gespräch oder beim Rundgang.
**Zweck:** Verstehen, wie Ware, Information und Verantwortung heute durch den Betrieb laufen — und wo es hakt.
**Haltung:** Zuhören und aufschreiben, nicht bewerten. Der Betrieb erklärt, wir dokumentieren. Kein Verkaufsgespräch während des Audits.

**Ausfüllen:** Pro Bereich die Leitfrage stellen, Beobachtung wörtlich notieren, dann Risiko und Bevero-Hebel einschätzen. Priorität = wie sehr drückt es **diesen** Betrieb, nicht wie schön der Use Case für uns ist.

---

## 1. Warenannahme

| Feld | Inhalt |
|---|---|
| Leitfrage | „Wie läuft eine typische Lieferung ab — von ‚LKW steht da' bis ‚Ware ist weggeräumt'? Wer nimmt an, wer prüft, wo wird was notiert?“ |
| Beobachtung | |
| Risiko | Fehlmengen und Qualitätsmängel fallen erst später auf; Lieferschein-Abweichungen sind nicht belegbar; Annahme hängt an einzelnen Personen. |
| Bevero-Hebel | Annahme mit Mengen, Abweichungen und Verantwortlichem festhalten — mobil, direkt an der Rampe. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## 2. Lagerorte

| Feld | Inhalt |
|---|---|
| Leitfrage | „Welche Lagerorte gibt es (Keller, Kühlhaus, Bar-Backup, Etagen) — und weiß jeder, was wo liegt?“ |
| Beobachtung | |
| Risiko | Suchzeiten; Doppelbestellungen, weil niemand weiß, was schon da ist; ‚Privatlogik' einzelner Mitarbeitender. |
| Bevero-Hebel | Lagerorte als klare Struktur abbilden — jede Bewegung hat einen Von- und einen Nach-Ort. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## 3. Warenbewegungen

| Feld | Inhalt |
|---|---|
| Leitfrage | „Wenn etwas vom Lager an die Bar oder in die Küche geht — wird das irgendwo festgehalten? Von wem?“ |
| Beobachtung | |
| Risiko | Schwund ist nicht von normalem Verbrauch unterscheidbar; Inventurdifferenzen ohne Erklärung; Diskussionen statt Daten. |
| Bevero-Hebel | Bewegungen zwischen Lagerorten und Ausgabestellen dokumentieren — nachvollziehbar, ohne Zettel. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## 4. Auffülllisten Bar / Küche

| Feld | Inhalt |
|---|---|
| Leitfrage | „Wie entsteht die Auffüllliste für Bar und Küche — Zettel, WhatsApp, Zuruf, Kopf? Und wie oft fehlt trotzdem etwas im Service?“ |
| Beobachtung | |
| Risiko | Ausverkaufte Positionen mitten im Service; Läufe ins Lager während der Stoßzeit; Frust zwischen Schichten („die Frühschicht hat wieder nicht aufgefüllt“). |
| Bevero-Hebel | Auffülllisten digital, mit Status: was gebraucht wird, was erledigt ist, wer es erledigt hat. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## 5. Schichtübergaben

| Feld | Inhalt |
|---|---|
| Leitfrage | „Wie erfährt die Spätschicht, was in der Frühschicht passiert ist — und was davon steht irgendwo geschrieben?“ |
| Beobachtung | |
| Risiko | Informationen gehen an der Schichtgrenze verloren; jeder erinnert sich anders; Neue und Aushilfen haben keinen Zugang zum ‚Flurfunk'. |
| Bevero-Hebel | Übergaben strukturiert festhalten: offene Punkte, Vorkommnisse, was die nächste Schicht wissen muss. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## 6. Freigaben / Korrekturen

| Feld | Inhalt |
|---|---|
| Leitfrage | „Wenn etwas korrigiert werden muss — falsche Menge, Bruch, Storno einer Entnahme — wie läuft das, und wer darf das entscheiden?“ |
| Beobachtung | |
| Risiko | Korrekturen passieren still; später weiß niemand, warum eine Zahl nicht stimmt; Freigaben hängen an Anwesenheit einer Person. |
| Bevero-Hebel | Korrekturen mit Grund und Freigabe dokumentieren — Änderungen bleiben sichtbar statt zu verschwinden. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## 7. Verantwortlichkeiten

| Feld | Inhalt |
|---|---|
| Leitfrage | „Wenn ich frage ‚Wer ist heute für die Warenannahme zuständig?' — bekommt jeder im Team dieselbe Antwort?“ |
| Beobachtung | |
| Risiko | Aufgaben fallen zwischen Rollen; ‚das macht doch immer der Marco'-Logik bricht bei Urlaub/Krankheit; Konflikte statt Klarheit. |
| Bevero-Hebel | Aufgaben haben einen sichtbaren Verantwortlichen und einen Status — nicht nur eine mündliche Absprache. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## 8. Transparenz für die Betriebsleitung

| Feld | Inhalt |
|---|---|
| Leitfrage | „Wenn Sie wissen wollen, was gestern am Standort passiert ist — wie lange dauert das, und wen müssen Sie dafür fragen?“ |
| Beobachtung | |
| Risiko | Überblick nur durch Hinterherlaufen; Entscheidungen auf Basis von Bauchgefühl; bei Problemen beginnt die Rekonstruktion bei null. |
| Bevero-Hebel | Ein Blick statt fünf Anrufe: Annahmen, Bewegungen, Übergaben und offene Aufgaben an einem Ort, mit Verlauf. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## 9. Medienbrüche

| Feld | Inhalt |
|---|---|
| Leitfrage | „Zählen wir mal durch: Wo im Warenfluss sind heute Papier, Excel, WhatsApp und mündliche Absprachen im Spiel?“ |
| Beobachtung (konkret auflisten: was läuft wo) | |
| Risiko | Jede Übergabe zwischen Medien ist eine Fehlerquelle; Informationen existieren mehrfach und widersprechen sich; nichts davon ist später auffindbar. |
| Bevero-Hebel | Die operativen Kernprozesse (Annahme, Bewegung, Auffüllen, Übergabe) laufen in einem Werkzeug — die bestehenden Systeme (Kasse, Buchhaltung, Bestellwesen) bleiben, wie sie sind. |
| Priorität | ☐ niedrig ☐ mittel ☐ hoch |

## Scorecard

Bewertung je Bereich: **1 = großes ungelöstes Problem … 5 = läuft sauber, kein Handlungsbedarf.**

| Bereich | Score (1–5) | Kurzkommentar |
|---|---|---|
| Warenannahme | | |
| Lagerorte | | |
| Warenbewegungen | | |
| Auffülllisten Bar/Küche | | |
| Schichtübergaben | | |
| Freigaben/Korrekturen | | |
| Verantwortlichkeiten | | |
| Transparenz Betriebsleitung | | |
| Medienbrüche | | |

**Lesart:**
- Mehrere Bereiche mit 1–2 → klarer Pilot-Kandidat; die zwei schwächsten Bereiche als Pilot-Scope vorschlagen.
- Überwiegend 3 → Pilot auf **einen** Prozess zuschneiden (z. B. nur Schichtübergabe oder nur Auffüllliste).
- Überwiegend 4–5 → ehrlich sagen: aktuell kein dringender Fall. Kontakt halten, nicht drängen.

**Nach dem Audit:** Ergebnis in 3–4 Sätzen an den Betrieb zurückspiegeln („Das habe ich verstanden — stimmt das so?“), bevor irgendein Angebot kommt.
