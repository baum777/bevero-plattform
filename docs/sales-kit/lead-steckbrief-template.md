# Lead-Steckbrief — Vorlage

**Zweck:** Vor einem Erstkontakt oder Erstgespräch in 15–20 Minuten verstehen, wie der Betrieb operativ tickt und ob Bevero dort einen echten Hebel hat.

**Grenze:** Nur öffentlich sichtbare Informationen (Website, Speisekarte, Google-Profil, Stellenanzeigen, Presse) oder Dinge, die im Gespräch freiwillig genannt wurden. Keine personenbezogene Tiefenrecherche, keine sensiblen Daten, keine Spekulation über einzelne Mitarbeitende.

| Frage | Eintrag |

## 6. Mögliche Pain Points

Ankreuzen, was plausibel ist — mit kurzer Begründung. Nicht mehr als 3 priorisieren.

| Pain Point | Plausibel? | Begründung |
|---|---|---|
| Übergaben zwischen Schichten gehen verloren (mündlich / Zettel / WhatsApp) | ☐ | |
| Bar-Auffülllisten oder Küchen-Checks laufen auf Papier oder Zuruf | ☐ | |
| Warenannahme wird nicht sauber dokumentiert (Mengen, Abweichungen) | ☐ | |
| Warenbewegungen Lager → Outlet sind nicht nachvollziehbar | ☐ | |
| Betriebsleitung hat keinen schnellen Überblick über den Standort | ☐ | |
| Korrekturen/Freigaben passieren informell und sind später nicht belegbar | ☐ | |
| Mehrere Standorte, keine einheitliche Prozessdokumentation | ☐ | |

## 7. Sicherer Mail-Aufhänger

Ein bis zwei konkrete Sätze, mit denen man Mail oder Gespräch eröffnen kann — bezogen auf **diesen** Betrieb, nicht generisch. „Sicher“ heißt: basiert auf einer öffentlich sichtbaren Beobachtung (Quelle in Abschnitt 5 notiert), unterstellt dem Betrieb kein Versagen und behauptet keine Produktfähigkeit außerhalb der Truth Table.

## 9. Risiken / No-Fit-Signale

| Signal | Vorhanden? | Kommentar |
|---|---|---|
| Betrieb zu klein (eine Person macht alles, kein Übergabe-Problem) | ☐ | |
| Gerade Großbaustelle (Umbau, Systemwechsel, Personalkrise) — kein Kopf frei | ☐ | |
| Erwartet ein vollwertiges Warenwirtschafts-/ERP-System — will Bestellwesen, Lieferantenpreise, Buchhaltung | ☐ | |
| Entscheidungswege unklar (Kette mit zentraler IT-Entscheidung woanders) | ☐ | |
| Kein erkennbarer Schmerz — Prozesse wirken bereits sauber gelöst | ☐ | |
